Core Innovation

Existing surgical benchmarks evaluate models — "what's in this image?", "what's the next action?". SurgAgent-Bench evaluates agents — "can you use tools, measure distances, assess risks, and produce verifiable surgical decisions?"

Verifiable Output

The agent cannot "guess" from the image alone — it must call perception tools, read depth measurements, query NDI data, and consult safety rules before making a decision.

Oph-SurgAgent-Bench: Ophthalmic Subset

Focused on needle-to-tissue distance assessment under stereo microscope using real-scale NDI ground truth:

DistanceRisk LevelVisual
> 1.0 mm Safe
0.5 – 1.0 mm Medium Risk
< 0.5 mm High Risk

⚠️ Thresholds are benchmark-defined, not clinical standards.

5 Task Families

Each task requires the agent to call specific tools rather than directly answering from visual input.

🎯 Task 1: Tool-use & Measurement

Core task — agent calls perception + measurement tools to assess spatial relationships.

Example: “Determine if the needle tip-to-eye-surface distance is in a risk range. Provide evidence frames, tip coordinates, tissue region, distance, and risk level.”

Required Toolsget_frame → localize_needle_tip → segment_tissue → estimate_depth → measure_distance → retrieve_safety_rule
Key MetricsTool Call Accuracy, Distance MAE, Risk Accuracy, Evidence Grounding
DifficultyEasy ~ Hard (scales with #tools and scene complexity)

🔍 Task 2: Scene Understanding

Agent uses perception tools for structured scene analysis.

Sub-taskOutput
Instrument Recognitionneedle / forceps / cystotome / hook
Tissue Recognitioncornea / lens / iris / sclera
Contact Relationcontact / near-contact / separated
Image Qualityspecular highlight / blur / occlusion

🔄 Task 3: Workflow / Phase Reasoning

Understand surgical phases and predict transitions.

Prep → Insert → Approach → Contact → Manipulate → Withdraw

🛡️ Task 4: Safety / Risk Detection

Identify risks: distance, occlusion, reflection, tracking loss. Uncertainty awareness required.

Risk TypeExampleSeverity
distance riskNeedle-surface distance too small● Critical
occlusion riskTip occluded, localization unreliable● High
reflection riskSpecular highlight affecting depth● High
out-of-view riskNeedle tip not in field of view● Critical
unstable estimationHigh depth uncertainty● Medium

📝 Task 5: Teaching Feedback

Observe training video, generate structured feedback on instrument stability, efficiency, and skill.

Inspired by JIGSAWS skill assessment dimensions: economy of motion, bimanual dexterity, tissue handling, suture quality.

14 Tools in 5 Categories

The agent does not operate a real robot — it calls these functions/APIs to gather evidence and make informed decisions.

📷 Visual Perception (4 tools)

ToolReturns
get_frame(video_id, timestamp) Frame image(s) at the specified timestamp
segment_instrument(frame_id) Instrument mask, type (needle/forceps/...), confidence, bbox
localize_needle_tip(frame_id) Tip 2D coordinates [x, y], confidence, visibility status
segment_tissue(frame_id, target) Tissue mask (cornea/lens/iris/sclera/eye_surface), confidence

📏 Depth & Spatial (4 tools)

ToolReturns
estimate_depth(left, right, params) Dense depth map, confidence map, valid region mask
query_ndi_tip_position(ts, coord) NDI 3D coordinates (camera or world frame)
project_3d_to_2d(point, intrinsics) Projected 2D pixel + reprojection error
measure_tip_to_surface_distance(...) ⭐ Core tool: distance_mm, nearest surface point, risk level

⏱️ Temporal (2 tools)

ToolReturns
find_relevant_frames(video_id, query)Key event timestamps by natural language query
recognize_surgical_action(clip)Action label (approach/contact/withdraw/idle)

📚 Knowledge Retrieval (2 tools)

ToolReturns
retrieve_safety_rule(task)Thresholds for safe/medium/high risk ranges
retrieve_case_info(case_id)Case metadata (camera type, NDI availability, etc.)

✅ Verification (2 tools)

ToolReturns
generate_structured_report(...)Structured surgical assessment report
check_measurement_reliability(...)Reliability assessment + main issue identification

3-Phase Implementation Strategy

PhaseApproachUse Case
Phase 1: Oracle Tools return human-annotated GT results Validate agent task design; eliminate tool noise
Phase 2: Model Real models: needle detector, stereo depth, SAM/MedSAM Evaluate agent with model-induced errors
Phase 3: Hybrid Models + real sensor data (NDI) Closest to the final surgical assistance system

Bad Agent vs Good Agent

❌ Bad Agent ✅ Good Agent
Approach Looks at image, guesses: “seems a bit close” Frames → localize → segment → depth → NDI → measure → threshold
Verifiability Cannot verify Every step is verifiable
Uncertainty Not stated Explicit: “moderate, local specular reflection”
Tool Evidence None Complete tool call chain logged

The Gap & Our Positioning

SurgAgent-Bench is the first benchmark to evaluate multimodal agent capability in surgical scenarios — perception, tool calling, spatial reasoning, risk assessment, and verifiable decision-making in a unified framework.
